1) NE（バックエンド）最終仕様

エンドポイント（本番）
	•	NE URL：固定の物

リクエスト（フラットJSON・階層ズレ無し）

必須：userId / text / mode / dstLang / requestId

{
  "userId": "<device-uuid>",
  "text": "こんにちは",
  "mode": "translate",
  "dstLang": "en",
  "requestId": "<uuid-v4>",
  "timestamp": 1734109400
}

レスポンス（鉄の掟：4要素固定）

{
  "decision": "GO|HOLD",
  "reason": "SUCCESS|BAD_LANG|BAD_MODE|NO_REQUEST_ID|TOO_LONG|EMPTY|ERROR",
  "source": "ZERO_API|CACHE|API",
  "result": "Hello"
}

防衛（要塞ロジック）
	•	requestId 必須（無ければ HOLD / NO_REQUEST_ID / ZERO_API）
	•	mode ホワイトリスト（translate/chat）→ それ以外は BAD_MODE
	•	translate時 dstLang ホワイトリスト（en/ja/zh/ko）→ 外れたら BAD_LANG
	•	空文 → EMPTY
	•	100文字超 → TOO_LONG
	•	成功（GO）のみキャッシュ保存、同一requestId再送は CACHE で即返し（原価ゼロ）
	•	source は原価の出所だけ（SpeakerやVersionと混ぜない）

⸻

2) クライアント（iOS/Flutter）最終仕様

画面：Translation History（ログ型翻訳）
	•	右：自分の入力（ユーザー発言）
	•	左：翻訳結果（シアン枠）→ タップで全画面拡大（相手に見せる）
	•	下部：入力欄＋呼吸するシアンマイク＋送信
	•	ボタン表記：GLB（動詞化）
実行中は PROCESSING…（スピナー禁止）

盾（Idempotency）
	•	_activeRequestId を成功するまで保持
	•	GO を受け取った時だけ _activeRequestId=null
	•	失敗/HOLD/通信エラー時は保持（連打・再送で CACHE が効く）

マイク権限（Robust Permission）
	•	Denied → request()
	•	PermanentlyDenied → openAppSettings()
	•	Grantedのみ listen
	•	赤禁止、シアン呼吸で状態表現
