1) 全体ロジック（要塞構造）

目的：会話AIにしない。前置きゼロで翻訳結果だけ返す。
主役：人間。AIは裏で“通すだけ”。
	•	入力（音声/テキスト）→ NE Gateway → 出力（翻訳結果）
	•	文脈保持・助言・人格・長文会話は やらせない
	•	失敗しても爆発しない：GO/HOLD＋理由固定＋原価管理（ZERO_API/CACHE/API） 

⸻

2) 画面導線（iOS/Flutter）

導線は固定：
main.dart → SplashScreen（演出）→ IstInterface（Translation History）

GLBボタンの出現条件

IstInterfaceの下部送信ボタン表示が
	•	_isLoading == false → GLB
	•	_isLoading == true → PROCESSING…

⸻

3) NE（バックエンド）最終仕様

本番URL（固定）：https://us-central1-tomori-world.cloudfunctions.net/tomoriRun

リクエスト（Flat JSON）

必須：userId / text / mode / dstLang / requestId（+timestamp） 

レスポンス（鉄の掟：4要素固定）

decision / reason / source / result  

要塞ルール（防衛）
	•	requestId必須（無ければHOLD/NO_REQUEST_ID/ZERO_API）
	•	mode whitelist（translate/chat）
	•	dstLang whitelist（en/ja/zh/ko など）
	•	100文字超 → TOO_LONG、空文 → EMPTY
	•	同一requestId再送はCACHEで即返し（原価ゼロ）  

⸻

(A) lib/main.dart（入口固定）
	•	home: const SplashScreen()

	•	言語切替（TO: ENGLISH / TO: JAPANESE）
	•	長押しで隠しコンソール（Release封印：kReleaseMode ? null : ...） 

(D) lib/screens/chat_screen.dart（隠しコンソール）
	•	mode=chatでNEへ（Releaseでは開けない前提） 

(E) lib/widgets/glowing_mic_button.dart（共通マイク部品）

7) 物流・医療へ転用できる理由（この部屋の核心）
	•	GLBは「翻訳アプリ」以上に、
**入力→判定→出力を止まらず返す“プロトコル”**を完成させた。
	•	requestId冪等性／GO-HOLD／source（原価管理）／即時応答
→ 物流・医療・インフラ入口にそのまま移植可能

⸻

	•	中身の“データ”は不要。仕様＋エンドポイント＋UI制約で動く（この部屋の設計がそれ）
