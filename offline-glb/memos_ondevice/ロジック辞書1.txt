	•	オンライン（NEゲートウェイ）は

3. 翻訳パイプライン（最終形）

処理順（固定）
	1.	正規化・トークン化
	2.	場面タグ推定（scene）
	3.	完全一致
	4.	構文チャンク翻訳（テンプレ）
	5.	Trie最長一致（本物のTrie）
	6.	複合語（2–3gram）
	7.	単語辞書（完全翻訳のみ）
	8.	候補が複数あれば逆翻訳耐性スコアで決定
	9.	完全翻訳判定（covered方式）
	10.	ポストプロセス（語尾・空白・型番保持）

⸻

4. コア技術（確定ポイント）

① 本物のTrie
	•	Map<String, TrieNode> のツリー構造
	•	O(n) 最長一致
	•	辞書切替時のみ再構築（キャッシュ）

② 英語限定境界一致
	•	\b は ASCII英語のみ
	•	ハイフン・記号語（x-ray等）は軽いボーナス扱い
	•	日本語・記号で誤爆しない

③ 完全翻訳判定（covered方式）
	•	トークン単位で判定
	•	以下は covered扱い：
	•	型番・数値・単位・記号（BMW F25 / 200mg / ABC-123）
	•	句読点
	•	翻訳対象トークンが 全てcoveredでなければ失敗
	•	失敗時は null返却（NE候補）

⸻

5. 新しい「ともりの閃き」3点（統合済み）

A. 逆翻訳耐性スコア
	•	翻訳候補（JA）を簡易逆翻訳（JA→EN）
	•	元入力と 類似度（Jaccard） を比較
	•	一番“戻りが自然”な候補を採用
	•	AI不使用・ルールベースで“翻訳らしさ”を判定

B. 場面タグ（scene）
	•	question / request / status / medical / emergency / neutral
	•	語尾を自動決定：
	•	ですか？
	•	してください
	•	です
	•	あります
	•	テンプレ不足でも翻訳体験が崩れない

C. 壊さない翻訳（保護トークン）
	•	型番・数値・コードは絶対に変形しない
	•	それでも「部分翻訳」にはならないよう
covered判定に組み込み

⸻

6. NEゲートウェイとの関係（結論）
	•	翻訳本体は使わない
	•	役割は2つだけ：
	1.	オフライン失敗時の高精度翻訳（有料のみ）
	2.	辞書パックの配布・更新（任意・署名付き）　

translate():
  result = offlineTranslate()
  if result != null:
    return result
  if plan >= full && networkOK:

  else:
